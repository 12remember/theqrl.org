Upcoming hard forks

The first planned QRL network hardfork is preliminarily announced for 25/02/2020, with a further suggested hard fork to follow once testing of on-chain-voting and on-chain governance through consensus changing voting proposals has been completed.

February 2020 hard fork feature list:

  • RandomX mining algorithm upgrade
  • Multisig addresses
  • Extended message transaction functionality
  • Ephemeral messaging stub functionality

Subsequent 2020 fork upgrade feature list:

  • Deterministic address support
  • Decentralised on-chain poll support (further testing required)
  • On-chain governance allowing vote-based hard fork changes to existing network parameters (further testing required).

Roadmap for future hard forks 2020/1 and beyond include:

  • Smart Contracts (Project Silicon)
  • Consensus mechanism change (PoS, dPoS)
